> AI가 코드를 짤 때 지켜야 할 규칙과 절대 하면 안 되는 것.
> 이 문서를 AI에게 항상 함께 공유하세요.
>
> **v2 변경 사항** (kkirikkiri 리뷰 반영): 테스트 명령어 버그 수정, 변호사 광고 표현 규칙 추가, 이미지 저작권·템플릿 고정·토큰 갱신 규칙 추가.

> **v2.1.3 변경 사항** (2차 콘텐츠 테스트 후 피드백): 캐릭터를 매 카드셋마다 완전히 동일한 이미지로 재사용하던 방식을 수정 -- 얼굴/의상/스타일은 고정하되 포즈·배경은 사건 소재에 맞춰 매번 새로 생성하는 것을 기본값으로 변경 (01_PRD.md 가정 원장 #19).

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 실행 환경 | Claude Code (로컬 스킬/스크립트) | 오늘 설치한 insane-search(유튜브/뉴스 크롤링), insane-design 등을 재사용할 수 있고, 별도 서버 호스팅이 필요 없음 |
| 소재 수집 | insane-search 플러그인 (yt-dlp, 뉴스/판결 사이트 접근) | 이미 설치되어 있고, 차단된 사이트도 우회 가능 |
| 콘텐츠 분석/요약 | Claude (수집된 원문을 직접 분석) | 별도 LLM API 키 없이 Claude Code 안에서 바로 실행 |
| 이미지 렌더링 | HTML+CSS 템플릿 -> Playwright로 PNG 캡처 | 기존 cardnews-maker의 HTML 카드뉴스 자산을 이미지로 승격시키는 가장 낮은 리스크 경로. 무료 |
| 캐릭터 일러스트 생성 | Gemini 이미지 API (`gemini-3.1-flash-image`, `scripts/generate_character.py`) | 웹툰풍 말풍선 없이 플랫 일러스트 캐릭터를 만들어 카드 여백을 채움. 하루 무료 한도 있고, 초과 시 선불 크레딧(prepay) 과금. **얼굴/의상/스타일은 `CHARACTER_STYLES`로 고정하고, 포즈·배경은 카드셋 소재에 맞춰 매번 새로 생성 -- 생성한 이미지는 `templates/characters/`에 계속 누적 저장** (01_PRD.md 가정 원장 #19) |
| Instagram 연동 | Instagram API with Instagram Login (`graph.instagram.com`) | Facebook 페이지 연결 없이 프로페셔널 계정만으로 캐러셀 게시 가능. 대용량 비디오 재개 업로드가 필요 없는 이미지 전용 프로젝트라 Facebook Login 방식이 필요 없음 |
| 이력 저장 | 로컬 JSON 파일 (`history/source_history.json`, `{type}:{source_key}` 키 객체) | 개인 도구 규모에 DB는 과함. git으로 이력도 함께 버전 관리 가능 |

## 프로젝트 구조

```
insta-cardnews-bot/
├── PRD/                        # 이 디자인 문서들
├── sources/                    # 수집된 원문(자막/기사/판결) 캐시
├── history/
│   └── source_history.json     # 중복 방지 이력 ({type}:{source_key} 키 객체)
├── templates/
│   ├── default_v1.html         # 승인된 고정 비주얼 템플릿 (임의 재생성 금지)
│   ├── characters/             # 카드 1 캐릭터 일러스트 (얼굴/의상 고정, 포즈·배경은 소재별로 새로 생성)
│   └── photos/                 # 실사 사진 자산 (예: lawyer-hanseyoung.jpg — 카드 7, 재생성 없이 고정 재사용)
├── fact-check/
│   └── {cardset_id}.md         # 카드별 핵심 주장 -> 원문 인용 위치 대조 자료
├── output/
│   └── {날짜}_{소재제목}_{cardset_id}/   # 생성된 PNG 카드뉴스 세트 (재생성 시 버전 보존)
├── scripts/
│   ├── generate_cardnews.*     # 소재 -> 카드뉴스 이미지 + 캡션 초안 + 원문대조 자료
│   ├── generate_character.py   # 카드 1 캐릭터 일러스트 생성 (카드셋마다 소재에 맞는 포즈/배경으로 실행, Gemini 이미지 API)
│   ├── mark_correction.*       # 게시 후 정정/삭제 상태를 이력에 기록
│   └── upload_instagram.*      # 승인된(잠긴) 세트 -> Instagram 게시
└── .env                        # Instagram API 토큰 등 (git 제외)
```

## 법률 광고 표현 규칙 (변호사 광고 규정 반영)

> 카드뉴스 문구, 캡션, CTA를 생성할 때 반드시 지켜야 하는 표현 규칙. 콘텐츠 분석/카드 구조화 단계에서부터 이 표를 기준으로 문구를 검토한다.

| 구분 | 규칙 |
|------|------|
| 금지 | "최고", "유일" 등 최상급·독점 표현 |
| 금지 | 승소율, 성공률 등 수치화된 성과 표현 |
| 금지 | 특정 사건의 결과를 모든 사건에 적용되는 것처럼 일반화·암시하는 표현 |
| 금지 | "OO 전문 로펌"처럼 로펌(법인) 단위로 전문분야를 표시하는 표현 |
| 허용 | "보험전문", "손해배상전문", "교통사고전문" 등 **개인 변호사**에게 등록된 전문분야 표현 |

> 카드 구조(후킹-쟁점-사례-법리-**결과**-팁-CTA)에서 "결과" 카드는 특히 이 규칙의 적용 대상이다. 매 카드뉴스마다 "개별 사건의 결과이며 모든 사건에 동일하게 적용되지 않는다"는 취지의 고지 문구를 CTA 근처에 포함한다.

## 캐릭터 일러스트 & 타이포 시스템 (v2.2)

> **v2.2 전환 (kkirikkiri 3인 냉정 평가 후 결정, 01_PRD.md 가정 원장 #21)**: 카드뉴스가 "AI가 만든 티가 난다"는 사용자 피드백에 대해 그로스/비주얼/카피 3개 관점으로 냉정 평가를 받은 결과, 원인의 대부분이 캐릭터 일러스트(반복되는 소품, 그라디언트 조명, 균일한 얼굴, 맥락에 안 맞는 미소)로 진단됐다. 이에 따라 **일러스트는 카드 1(훅)에만 남기고, 카드 2~7은 전부 타이포/숫자 중심으로 전환**했다. 아래는 이 결정을 반영한 최신 규칙이다 (이전 버전에서 "모든 카드에 캐릭터 배치"라고 했던 규칙은 폐기됨).

| 카드 | 비주얼 요소 |
|------|-------------|
| 1 (훅) | 캐릭터 일러스트 (의뢰인, 걱정스러운 표정) — 유일하게 일러스트가 남은 카드. 캠페인마다 시간대·장소·구도를 다르게 (v2.2.4) |
| 2 (쟁점) | 타이포만, 여백을 그대로 살림 (배경 워터마크는 v2.2.4에서 제거) |
| 3 (사례) | `.hero-stat`으로 사건의 핵심 숫자(금액/거리/기간 등)를 크게 반복 노출 |
| 4 (법리) | 번호 매긴 `.legal-list`, 여백을 그대로 살림 (배경 워터마크는 v2.2.4에서 제거) |
| 5 (결과) | `.stat-compare-hero` (before/after 비교를 확대) |
| 6 (팁) | `.tip-list` (체크마크 3개짜리 행동지침 목록, v2.2.4 -- 이전의 `.pull-quote` + 배경 워터마크에서 교체) |
| 7 (CTA) | `.lawyer-photo`로 한세영 변호사 실사 사진, 흰색 프레임 + 파란 배경 (v2.2.1, v2.2.4 -- 01_PRD.md 가정 원장 #22, #24) |

- **배경 워터마크(`.giant-mark`)는 v2.2.4에서 전면 폐지**: kkirikkiri 재평가에서 "크고 흐린 배경 글자/기호"가 그 자체로 AI 랜딩페이지 생성기의 전형적 클리셰라는 지적을 받았다. 카드 2/4는 제거하고 여백을 그대로 두고, 카드 6은 `.tip-list`(체크마크 목록)로 대체했다 -- **빈 공간은 장식이 아니라 실제 정보(체크리스트, 숫자)로 채운다**는 원칙. `.giant-mark` 클래스 자체는 CSS에 남아있지만 더 이상 마크업에서 쓰지 않는다.
- **카드 1 일러스트만 유지, 프롬프트도 손질**: `scripts/generate_character.py`의 `BASE_STYLE`에 그라디언트/은은한 조명 금지, 화분·머그컵 등 반복 소품 최소화 지시를 추가했다. 카드 1은 여전히 `CHARACTER_STYLES`로 얼굴/의상을 고정하고, 사건별로 배경/포즈만 새로 생성한다 (01_PRD.md 가정 원장 #19 원칙 유지, 카드 1에 한해서만 적용).
- **캠페인마다 카드 1 장면을 실제로 다르게 (v2.2.4)**: 두 캠페인이 "책상+스탠드+걱정스러운 표정"으로 똑같이 나오면 "찍어낸 스톡컷" 인상을 오히려 강화한다는 지적을 받았다. 시간대(주간/야간), 장소(부엌 식탁/거실 소파), 구도(클로즈업/와이드샷), 소품(머그컵/책장)을 캠페인마다 의도적으로 다르게 프롬프트에 명시한다.
- **캐릭터 프롬프트 실패 사례 2가지 (v2.2.4, 재발 방지용 기록)**: (1) 편지 소품에 "INSURANCE LETTER", "REJECTION" 같은 **영어 텍스트가 그대로 렌더링된 적**이 있다 -- 프롬프트에 "the letter shows only abstract blurred line placeholders for text (no readable words, no English letters, no legible text anywhere in the image)"처럼 명시적으로 못 박을 것. (2) 한 장면을 요청했는데 **2컷으로 분할된 이미지(diptych)**가 나온 적이 있다 -- "a single unified illustration (not a diptych, not a comparison grid, no panel divisions, one continuous scene only)"를 프롬프트 앞부분에 넣을 것.
- **생성 후 드리프트 확인 필수**: Gemini는 같은 프롬프트 접미사를 줘도 의상 색상·무늬, 안경 유무, 수염 유무 등이 매번 미세하게 달라질 수 있다. 생성 직후 반드시 기존 캐릭터 이미지와 육안으로 비교하고, 어긋나면 프롬프트에 "plain solid navy blue sweater, no stripes", "no glasses, clean-shaven"처럼 구체적 제약을 추가해 재생성한다.
- **`.giant-mark`/`.firm-mark` 등 배경 장식은 반드시 `<span>`으로 작성**: `<div>`로 쓰면 `.card > div { position: relative; z-index: 1; }` 규칙과 충돌해 `position: absolute`가 깨지고 레이아웃이 무너진다 (v2.2 개발 중 실제로 발생한 버그 — 처음 만들었던 `.firm-mark`가 이 문제로 안 보이는 채로 렌더링됐었다).
- **카드 7 실사 사진 (v2.2.1, v2.2.4)**: `templates/photos/lawyer-hanseyoung.jpg`가 한세영 변호사의 고정 프로필 사진이다. `.lawyer-photo`(340×440px, `object-position: 50% 15%`, 흰색 10px 프레임)로 표시하며, 카드 1 캐릭터 일러스트와 달리 **카드셋마다 재생성하지 않는 고정 자산**이다. `templates/_preview/characters/`처럼 `templates/_preview/photos/`에도 동일 파일을 복사해둬야 미리보기 렌더링에서 경로가 풀린다. 카드 7 배경은 흰색(`--surface`)이 아니라 카드 1·5와 같은 연한 파란색(`--brand-soft`)으로 고정 -- 실사 사진과 나머지 플랫 카드 사이의 톤 차이를 완화하고, 카드 1(훅)과 카드 7(CTA)이 같은 색으로 시작·끝을 감싸는 북엔드 효과를 준다.
- **카드 7 BODY_7은 짧게, 한 줄에 들어가도록 (v2.2.3)**: `.card-7 .body-text { max-width: 600px; }`로 좁혀져 있다 (사진이 우측 하단 x:684~를 차지하므로 기본 900px 너비로는 텍스트가 사진 위로 올라감). 좁은 폭이라 문장이 길면 2~3줄로 쪼개지면서 마지막 줄만 짧게 남아 "글자 배치가 안 예쁜" 느낌을 준다 -- BODY_7은 "버리지 마세요. 제가 확인해드릴게요." 정도로 한 줄(15자 내외)에 들어가는 짧은 문장으로 쓴다. 1인칭 화법(v2.2.2)을 넣더라도 부연설명을 욕심내지 말고 짧게 끊을 것.
- **도와준다는 인상**: 카피에 "~이 짚어주는", "~와 함께" 같은 표현을 일부 카드(법리·결과·팁)에 자연스럽게 녹여, 실제로 문제 해결을 도왔다는 느낌을 전달한다. 단, 모든 카드에 억지로 넣지는 않는다 (후킹·쟁점·사례는 의뢰인 시점 그대로 유지).
- **카드 5(결과)는 헤드라인을 담백하게**: "~와 함께 뒤집은/받아낸 ~" 처럼 기여를 헤드라인에 크게 내세우면 과장된 느낌을 준다 (01_PRD.md 가정 원장 #18). 헤드라인은 사건 결과 자체(예: "혐의없음으로 종결된 사건", "재심사 청구로 뒤집힌 결과")로 쓰고, "~와 함께 대응한 사건입니다" 문장은 카드 하단 `.disclaimer-tag`의 짧은 도입부로만 넣는다.
- **개인 변호사명 vs 법인명 (v2.1.4)**: 카드 4(법리)의 헤드라인과 카드 5(결과)의 `.disclaimer-tag` 도입부는 **"법무법인 한앤율"**(법인명)을 주어로 쓴다. 반면 각 카드 하단의 `.footer-legal`("광고책임변호사 : 한세영 · 1533-0854")과 카드 6의 "한세영 변호사의 TIP" 배지, 카드 7의 `.lawyer-info`는 **"한세영"**(개인 변호사명)을 그대로 유지한다 -- 광고책임변호사 표시는 법률상 개인 단위로 하는 표시이기 때문. 법인명 사용이 "OO 전문 로펌"식 로펌 단위 전문분야 표시(금지 항목)가 되지 않도록, 법인명 뒤에는 "전문" 등 전문분야를 암시하는 단어를 붙이지 않는다.
- **`.hero-stat`/`.stat-compare-hero`는 값을 짧게**: 72~108px 대형 숫자 폰트에서 괄호 딸린 부연설명("0원 (지급거절)")을 넣으면 줄바꿈이 어색해진다. 값은 숫자/짧은 명사만 넣고, 부연설명이 필요하면 `.stat-label`(작은 글씨) 쪽에 넣는다.
- **`.hero-stat`은 라벨이 숫자 위에 (v2.2.5)**: 마크업 순서는 `<span class="label">`이 먼저, `<span class="value">`가 다음이다 (KPI 타일 관례 -- `.stat-compare`의 `.stat-label`/`.stat-value` 순서와 동일하게 맞춤). `margin-top: 48px`로 위 `.case-quote` 박스와의 간격도 넉넉히 뒀다 (기존 32px는 붙어 보인다는 지적).
- **1인칭 변호사 목소리 (v2.2.2)**: 카드 6(팁)·카드 7(CTA)의 본문은 "버리지 마세요", "확인해보세요" 식 안내문 톤 대신 "제가 직접 검토해드립니다", "제가 먼저 살펴봐드리겠습니다"처럼 1인칭으로 쓴다 -- 변호사 본인이 말을 거는 느낌을 주기 위함 (01_PRD.md 가정 원장 #23). 카드 4(법리)는 법인명 주어(v2.1.4)를 유지하므로 1인칭을 넣지 않는다. **"제가"가 줄 끝에서 쪼개지지 않는지 렌더링해서 확인** -- `제\n가`처럼 대명사가 줄바꿈되면 어색하니, 문장을 짧게 끊어서 회피한다.
- **헤드라인 숫자 앞세우기 (v2.2.2)**: 카드 1(훅) 헤드라인은 가능하면 사건의 핵심 숫자를 문장 맨 앞에 놓는다 (예: "850만원 보험금, '기왕증'이라며 거절당했습니다", "병원까지 90km, 이 거리가 보험사기가 될까요?"). 뒤 카드(3, 5)에서 같은 숫자가 다시 나와도 반복 노출은 의도된 것 -- growth 리뷰어가 지적한 "숫자가 5페이지까지 안 나온다"는 문제에 대한 대응.
