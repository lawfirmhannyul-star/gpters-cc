## kkirikkiri 리뷰로 바뀐 것 (v1 -> v2)

3명의 리뷰어(기술/데이터, Phase/법률리스크, 놓친 부분)가 검토한 결과를 전부 반영했습니다.

- **캡션/원문대조 자료가 파이프라인에 추가됨**: 이미지만이 아니라 캡션 초안과 "카드별 주장 -> 원문 근거" 대조 자료도 함께 생성·저장됩니다.
- **승인이 디자인/사실관계로 분리됨**: `design_approved`와 `fact_check_confirmed`를 각각 확인해야 게시가 가능합니다 (02_DATA_MODEL.md CardSet 참고).
- **변호사 광고 표현 규칙이 명문화됨**: 최상급/승소율/일반화/로펌단위전문 표현 금지, 개인 등록 전문분야("보험전문" 등) 표현은 허용 (04_PROJECT_SPEC.md 참고).
- **의뢰인 보호 장치 추가**: 사건 진행상태(`case_status`)와 의뢰인 동의확인(`consent_confirmed`)이 없으면 소재로 못 씀.
- **이미지 저작권 규칙 추가**: 라이선스 프리/자체 제작 이미지만 사용.
- **게시 후 정정/삭제 이력 추가**: `UploadRecord.status`에 `corrected`/`deleted` 추가.
- **테스트 명령어 버그 수정**: `graph.facebook.com` -> `graph.instagram.com`으로 정정 (2명의 리뷰어가 독립적으로 발견).
- **토큰 갱신, 소재 부적합 처리, 성과 지표 추적**이 Phase 2~3에 추가됨.

## 사용자 디자인 피드백으로 바뀐 것 (v2 -> v2.1)

모바일(인스타그램) 미리보기로 실제 화면을 확인한 뒤 받은 피드백을 전부 반영했습니다.

- **8장 -> 7장으로 축소**: 마지막 "요약" 카드가 법리 카드와 내용이 중복돼 제거함.
- **캐릭터 일러스트 도입**: 처음엔 웹툰풍 캐릭터를 배제했었는데(아래 "확인된 것" 참고), 실제 모바일 화면에서 그림 없는 카드가 너무 밋밋하다는 피드백에 따라 방향을 바꿈. 말풍선 없는 플랫 일러스트 스타일로 "의뢰인"/"한세영 변호사" 2개 고정 캐릭터를 Gemini 이미지 API로 생성, 모든 카드에 배치.
- **"변호사가 도와줬다"는 인상 강화**: 결과 카드는 단순히 "재심사 청구로 뒤집힘"이 아니라 한세영 변호사와 의뢰인이 함께 있는 그림 + "한세영 변호사와 함께 뒤집은 결과" 문구로 변경. 법리/팁/CTA 카드도 변호사 캐릭터가 등장.
- **캐릭터 자산은 재사용 원칙**: 한 번 생성한 캐릭터 이미지는 `templates/characters/`에 고정 저장하고, 카드셋마다 재생성하지 않음 (04_PROJECT_SPEC.md "캐릭터 일러스트 시스템" 참고).

## 비주얼 템플릿 확정 (v2.1)

`templates/default_v1.html` — 토스(toss.im)를 레퍼런스로 실측 분석해서 만든 7장 카드뉴스 템플릿입니다. `insane-design/toss/design.md`에 분석 근거가 있습니다.

- 플랫 일러스트 캐릭터 2종("의뢰인", "한세영 변호사")이 모든 카드에 등장합니다. 웹툰풍 말풍선은 쓰지 않았습니다 (카드 정보성·신뢰성 톤 유지, 결과 카드의 광고규정 고지 문구와의 톤 충돌 방지).
- 7장 모두 하단에 `광고책임변호사 : 한세영 · 1533-0854` 표시 포함 (카드 낱장이 단독으로 공유될 수 있어 전체 카드에 반영).
- Instagram 핸들 `@hannyul_official` 반영.
- 유튜브 채널 링크는 카드 이미지 안에서 클릭되지 않습니다 (인스타그램 피드 이미지는 하이퍼링크 미지원 — 플랫폼 제약). 채널명은 텍스트로만 표시, 실제 링크는 프로필 bio에 둬야 합니다.
- 미리보기: `templates/_preview/index.html` (예시 사건으로 채운 7장), `templates/_preview/screenshots/card-1~7.png` (실제 캡처본)
