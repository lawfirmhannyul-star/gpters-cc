## 1. 제품 개요

### 한 줄 요약
유튜브 영상, 보험 관련 뉴스/판결 링크, 직접 처리한 사건 사례를 소재로 삼아 인스타그램 카드뉴스 이미지를 자동 생성하고, 변호사 본인이 디자인과 사실관계를 각각 승인하면 인스타그램에 업로드하는 Claude Code 기반 반자동 파이프라인.

### 해결하는 문제
한세영 변호사가 지금 겪는 병목은 세 가지다.
1. 카드뉴스 이미지를 직접 만들어본 적이 없다 (디자인 역량 부재)
2. 예전에 외주를 맡겼지만 법률 정보를 정확히 요약하지 못했다 (품질 문제)
3. 완성된 콘텐츠도 시간이 없어 직접 업로드하지 못한다 (실행 병목)

→ 소재만 있으면(또는 소재가 없어도) 이미지·캡션·원문대조 자료 생성까지는 완전 자동, 업로드는 "확인 후 승인"만 하면 되는 구조가 필요하다.

### 핵심 가치
- **품질 고정**: 정보 요약을 Claude가 원문(자막/기사/판결문) 기반으로 수행하고, 원문 대조 자료를 함께 제공 -- 사람이 임의로 손대며 생기는 왜곡을 줄인다
- **이중 승인 게이트**: 자동이지만 무인은 아니다. "디자인이 보기 좋은가"와 "사실관계가 맞는가"를 분리해서 확인받는다 (변호사 광고 규정과 신뢰도를 고려한 설계)
- **소재 걱정 해소**: 소재를 안 줘도 유튜브 채널에서 아직 안 쓴 영상을 스스로 찾아온다
- **속도**: 소재 확보 -> 검토 준비 완료까지 사람이 손댈 일이 거의 없다

## 2. 사용자

### 주요 사용자
- **누구**: 한세영 변호사 (보험 전문, 유튜브 채널 "한세영변호사tv" 운영, 본인 전용 도구)
- **상황**: 유튜브 영상을 올린 직후, 보험 관련 뉴스/판결을 접했을 때, 또는 시간이 날 때 SNS에 뭔가 올리고 싶을 때
- **목표**: 유튜브 콘텐츠와 실무 경험을 인스타그램 팔로워 성장과 신뢰 축적으로 연결

### 사용자 시나리오
1. 변호사가 유튜브 영상 URL(또는 뉴스/판결 URL, 또는 직접 사건 요약 텍스트)을 도구에 전달하거나, 아무것도 주지 않는다
2. 소재가 없으면 도구가 유튜브 채널에서 아직 다루지 않은 최신 영상을 스스로 찾아 제안한다
3. 도구가 소재를 분석해 카드뉴스 이미지 세트(PNG) + 캡션 초안 + 원문 대조 자료를 만들어 로컬에 저장한다
4. 변호사가 이미지·캡션·대조 자료를 확인하고, 디자인과 사실관계를 각각 승인한다
5. 도구가 인스타그램 계정에 캐러셀 게시물을 올리고, 사용한 소재를 이력에 기록해 다음에 중복되지 않게 한다
6. 게시 후 오류를 발견하면 변호사가 정정/삭제 사실을 도구에 알려 이력에 남긴다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
소재 확보(입력 또는 자동탐색+중복체크) -> 콘텐츠 분석/요약 -> 카드뉴스 이미지+캡션+원문대조 생성 -> 로컬 미리보기 -> 디자인 승인 + 사실관계 승인 -> Instagram 업로드 -> 이력 기록 -> (필요시) 정정/삭제 기록
```

### 상세 흐름
1. **소재 확보**: 유튜브 URL/뉴스 URL/판결문 URL/텍스트를 입력받거나, 미입력 시 유튜브 채널의 최신 영상 중 이력에 없는 것을 자동 선정. 텍스트 소재는 사건 진행상태(확정/진행중)와 의뢰인 동의 확인 여부를 함께 확인.
2. **콘텐츠 분석**: Claude가 원문(자막/기사/판결문/텍스트)을 읽고 카드별 제목·본문·이미지 키워드를 결정, 법률 정보 정확성과 광고 표현 규칙을 최우선으로 요약
3. **생성**: 승인된 고정 템플릿으로 1080x1080 카드 세트를 PNG로 캡처하고, 캡션/해시태그 초안과 원문 대조 자료를 함께 생성
4. **검토**: 로컬 폴더에 이미지·캡션·대조자료 저장, 변호사가 직접 확인
5. **승인**: 변호사가 디자인 승인과 사실관계 승인을 각각 확인 -- 둘 다 완료되면 세트가 잠김(locked)
6. **업로드**: 잠긴 세트만 Instagram API로 캐러셀 게시 (미디어 컨테이너 생성 -> 게시)
7. **기록**: 사용한 소재(영상 ID/URL)와 게시 결과를 이력 파일에 저장
8. **사후 대응**: 게시 후 오류를 발견하면 변호사가 정정/삭제 사실을 알리고, 도구가 이력에 반영

## 5. 성공 기준

- [ ] 유튜브 URL 하나를 입력하면 카드뉴스 이미지 + 캡션 + 원문 대조 자료가 자동으로 생성된다
- [ ] 소재를 입력하지 않아도 유튜브 채널에서 아직 안 쓴 영상을 스스로 찾아 제안한다
- [ ] 동일한 영상/기사/판결을 두 번 소재로 쓰지 않는다 (중복 방지 이력이 실제로 작동)
- [ ] 카드 문구가 원문과 대조된 상태로 변호사가 사실관계를 확인(fact_check_confirmed)한 뒤에만 게시된다
- [ ] 변호사가 디자인·사실관계 승인을 모두 마치면 실제로 인스타그램 계정에 캐러셀 게시물이 올라간다
- [ ] 게시 후 오류를 발견했을 때 정정/삭제 사실을 이력에 남길 수 있다
- [ ] 게시물의 게시일·소재 유형이 기록되어, 이후 어떤 소재/형식이 팔로워 성장에 도움이 됐는지 주기적으로 되짚어볼 수 있다

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 완전 무인 자동 업로드(승인 없이 스케줄러가 게시) -- 이유: 변호사 광고 규정상 검수 없는 게시는 리스크가 크고, Meta 앱 심사(2~4주)와 별개로 신뢰 문제가 생길 수 있음. Phase 3에서 조건부 재검토.
- 릴스(영상) 콘텐츠 자동 생성 -- 이유: 이번 요청은 카드뉴스(이미지) 파이프라인에 집중, 영상 편집은 범위 밖
- 다계정/다른 변호사용 멀티테넌시 -- 이유: 본인 전용 도구
- 댓글 자동 응답, DM 자동화 -- 이유: 요청 범위 밖
- 성과 지표의 자동 집계/대시보드화 -- 이유: Instagram Insights를 수동으로 참고하는 것으로 Phase 1~2는 충분, 자동 집계는 Phase 3 이후 검토
- Meta 앱 리뷰 대행, 변호사 광고 규정 법률 검토 대행 -- 이유: 도구가 아니라 사람이 처리해야 하는 행정/법적 절차
